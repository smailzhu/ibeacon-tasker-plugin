# Security Policy

## Reporting A Vulnerability

Please report security issues privately before opening a public issue. If the
repository owner has enabled GitHub private vulnerability reporting, use that
flow. Otherwise, open a minimal public issue asking for a private contact path
without disclosing exploit details.

Include:

- affected version or commit
- Android version and device model, if relevant
- clear reproduction steps
- impact and any known workaround

## Signing Keys And Secrets

Do not report signing keys, keystores, passwords, or release secrets in public
issues, pull requests, logs, screenshots, or attachments. If key material is
ever exposed, treat it as compromised and rotate it before publishing another
release.

## Supported Versions

During the initial public release period, security fixes are handled on the
latest tagged release and current `main`.
